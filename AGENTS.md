# 绝对规则

- **语言**：推理内容（Thought）与对话内容必须使用中文。代码、变量名、API 字段保留英文，注释用中文。
- **搜索源**：Web 拉取时优先检索国内可达的中文源（如百度百科、知乎、CSDN 等）。
- **指令纪律1**：严格按用户指令执行。build 模式下若方向未明确，先问再动；plan 模式下绝不动手。
- **指令纪律2**：禁止主动执行可能影响用户判断或较复杂的操作（例如 git 指令、前后端联调 等）。
- **指令纪律3**：在wsl环境下执行不了的操作优先转向windows下执行，如仍遇异常则与用户协商。
- **执行边界1**：包导入测试、test/测试只在必要时候（如明确修改相应代码需验证/用户需求时）执行。
- **执行边界2**：查看代码时发现阻塞性 bug 等问题先与用户一同商议对策再进行操作。

# 项目：TravelPal

旅行伴侣 —— 基于双引擎 + LLM Agent

## 代码规范

写代码时遵循 docs/coding_standards.md 中的接口清单规范与注释规范。

## 双终端协作

- **Alpha**：负责代码实现、架构变更、算法优化。执行具体的编码任务。
- **Bravo**：负责文档同步、架构记录、总结反思。维护 md 文件与规范。

### 握手协议

1、任一方均可给对方派发任务。relay-chat 调用失败时输出 "⚠️ 信息未传递，请检查 relay 连接"。
2、用户直接向某一方派发的任务，执行方完成后无需通知另一方。

派发任务完整工作流:
```
① 派发方 → relay-chat(执行方, 任务描述)
② 执行方完成 → relay-chat(派发方, ✅ 完成通知)
③ 验收不通过 → relay-chat(执行方, "仍需修正：<说明>") → 回到 ②
```
