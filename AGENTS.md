## 代码规范

写代码时遵循 docs/coding_standards.md 中的接口清单规范与注释规范。

## 双终端协作

- **Alpha**：负责代码实现、架构变更、算法优化。执行具体的编码任务。
- **Bravo**：负责文档同步、架构记录、总结反思。维护 md 文件与规范。

### 握手协议

任一方（Alpha/Bravo）均可给对方派发任务和发送完成通知。
relay-chat 调用失败时输出 "⚠️ 信息未传递，请检查 relay 连接"。

```
① 派发任务
   派发方 → relay-chat(执行方, 任务描述)
   执行方 → "✅ 已接收任务：<摘要>"

② 完成任务
   执行方 → relay-chat(派发方, 完成通知)
   派发方 → "✅ 已接收完成通知"

③ Review 核验
   派发方检查结果:
   ├── 通过 → relay-chat(执行方, "任务已通过")
   │         执行方 → "✅ 任务已核验"
   └── 不通过 → 修正流程 ↓

④ 不通过修正（简化版）
   修正方修改完结果后 → relay-chat(派发方, "已修正：<说明>")
   派发方审查:
   ├── 通过 → relay-chat(修正方, "任务已通过")
   │         修正方 → "✅ 任务已核验"
   └── 不通过 → relay-chat(修正方, "仍需修正：<说明>")
             修正方 → 继续修正后回到 ④ 开头
```
