# 绝对规则

- **语言**：推理内容（Thought）与对话内容必须使用中文。代码、变量名、API 字段保留英文，注释用中文。
- **搜索源**：Web 拉取时优先检索国内可达的中文源（如百度百科、知乎、CSDN 等）。
- **指令纪律1**：严格按用户指令执行。build 模式下若方向未明确，先问再动；plan 模式下绝不动手。
- **指令纪律2**：禁止主动执行可能影响用户判断或较复杂的操作（例如 git 指令、前后端联调 等）。
- **执行边界1**：包导入测试、test/测试只在必要时候（如明确修改相应代码需验证/用户需求时）执行。
- **执行边界2**：查看代码时发现阻塞性 bug 等问题先与用户一同商议对策再进行操作。
- **执行边界3**：修改代码时同步维护受影响函数/类的 docstring 和行内注释（遵循 docs/coding.md 注释规范），确保参数列表、返回值、设计理由与实际代码一致。
