# 绝对规则

## 指令纪律

- **纪律1**：严格按用户指令执行。build 模式下若方向未明确，先问再动；plan 模式下绝不动手。
- **纪律2**：禁止主动执行可能影响用户判断或较复杂的操作（如 git commit/push/pull/merge/checkout、前后端联调等）。git 操作严格遵循 [`docs/git.md`](docs/git.md) 分支策略。
- **纪律3**：新对话开始时，若用户未提供上下文，主动询问是否需要回顾当前项目状态或关键决策。不自作主张假设已知上下文。

## 执行边界

- **边界1**：包导入测试、test/测试仅在以下情况执行：
  - 用户明确要求运行测试
  - 修改了测试代码本身需要验证
  - 用户要求验证某功能是否正常
  - 执行时仅运行受改动的模块对应的测试，不运行无关测试
- **边界2**：查看代码时发现阻塞性 bug 等问题先与用户一同商议对策再进行操作。
- **边界3**：修改代码时同步维护受影响函数/类的 docstring 和行内注释（遵循 [`docs/coding.md`](docs/coding.md) 注释规范），确保参数列表、返回值、设计理由与实际代码一致。

## 通用约束

- **语言**：推理内容（Thought）与对话内容必须使用中文。代码、变量名、API 字段保留英文，注释用中文。
- **搜索源**：Web 拉取时优先检索国内可达的中文源（如百度百科、知乎、CSDN 等）。
