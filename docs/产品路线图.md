 # 产品路线图

## 第一阶段：前期 / MVP 阶段

### 目标简介
实现「一次对话 + 一次规划」的核心闭环，验证 VNS+ 引擎与 LLM 对话的协作模式。

对应 agent-MVP.md Phase 1 + Phase 2 + Phase 3。

### MVP 边界
只做行前共创规划，不做行中实时调整和行后回顾。

### 包含内容

| 项 | 说明 | 对应 Phase | 关键约束 |
|----|------|-----------|----------|
| LLM 营业时间解析 | 替代纯函数解析高德 opentime2 | Phase 1 | 先规则后 LLM，规则失败再调 API，不阻塞整体进度 |
| Commentator 评语生成 | 规划完成后自动生成自然语言解说 | Phase 1 | 规则模板 + LLM 润色混合模式，降低调用成本 |
| SSE Chat 接口 | `POST /api/chat` 流式输出，结构化消息 `{type, content, metadata}` | Phase 1 | 先 Mock Agent 再替换真实 LLM，前后端可并行 |
| AgentPage.vue | 独立聊天页面，输入框 + 消息列表 + SSE 流式渲染 | Phase 2 | 无 |
| ChatMessage.vue | 单条消息组件（打字机效果） | Phase 2 | 无 |
| HistoryPage | localStorage 存储最近 10 次规划 | Phase 2 | ~20 行代码，低成本高完成感 |
| Planner 三指令 | 加景点 / 去景点 / 调天数，直接调引擎重算 | Phase 3 | LLM 只做意图识别+参数提取（Function Calling 定死 3 意图），不碰路径计算 |

### 技术验证

- SSE 流式输出稳定性 + 结构化消息格式
- VNS+ 引擎在对话场景下的实时重算能力
- 高德 API 对接（营业时间 LLM 解析、实时路况）

## 第二阶段：中期 / 求职完善期

### 目标简介
完善产品体验，积累叙事素材，形成求职时可展示的完整故事。

### 交互锚点
对话式聊天作为首页主交互（非表单填写），聊天面板与地图左右联动，用户说需求、后端出方案。

### 包含内容

| 项 | 说明 | 对应 Phase | 关键约束 |
|----|------|-----------|----------|
| Active Info Push | 规划完成后主动推送天气/高铁/热门事件 | Phase 1（延期） | 和 MCP 一起落地 |
| 三视图联动 | 左：行程列表、中：AMap 地图、右：Agent 聊天，联动交互 | Phase 4 | 点击景点→地图定位+Agent 解说，三面板实时同步 |
| 方案对比 | 选 2-3 个方案对比总成本/距离/时间等指标 | Phase 4 | 轻量 CSS/SVG 图表，不用第三方库 |
| 知识问答演示 | ChromaDB + 景点知识库，Agent 可回答景点相关问题 | Phase 4 | RAG 能力精简演示，视觉展示有限 |
| 历史记录集成 | 历史方案在详情页中可快速回顾 | Phase 4 | 依赖 Phase 2 的 HistoryPage 基础 |

### 六个完善方向

| 方向 | 内容 | 价值 |
|------|------|------|
| 主动信息补充 | 出发前主动查询天气、高铁时长、热门事件 | 体现「主动陪伴」的产品理念 |
| 过程可视化 | 展示 VNS 搜索进度（"正在为你寻找最优路线..."） | 增加沉浸感和情感反馈 |
| 实时路况整合 | 接入高德实时拥堵 API，标注避堵路线 | 提升方案的实用可信度 |
| 多方案对比 | 同时展示 2-3 条路线，横向对比指标 | 保留用户决策感 |
| 无障碍 / 家庭友好 | 增加可筛选的"无障碍友好""亲子"标签 | 体现产品 sense |
| PDF 旅行手册导出 | 一键生成可分享的行程手册 | 社交传播 + 实用价值 |

### 叙事亮点
核心一句话：「我的项目不是做一个更好的规划工具，而是做一个有温度的旅行伴侣。它不替用户做决定，只在用户需要的时候提供可靠的支持。」

## 第三阶段：后期 / 理想目标

### 目标简介
实现全流程智能旅行伙伴——从行前灵感、行中兜底到行后复盘。

### 五个体验场景

| 场景 | 描述 |
|------|------|
| 行前共创规划 | 用户说想法，系统补信息、给建议，边聊边完善方案 |
| 行中实时调整 | 用户说"想多睡会儿"，自动重算后续路线和时间窗 |
| 主动智能推荐 | 基于用户偏好和实时数据，主动推荐附近景点、美食 |
| 事后出行复盘 | 生成旅行回忆录、足迹地图、分享卡片 |
| 记忆与复访 | 记住用户偏好、旅行风格，每次打开都是老朋友 |
