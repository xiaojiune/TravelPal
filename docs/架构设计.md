# TravelPal 架构设计

## 一、概述

TravelPal 是一个基于**双引擎 + LLM Agent** 的智能旅游路径规划系统。用户通过自然语言描述出行需求，系统自动调用高德 API 获取真实 POI + 驾车数据，经 SA + VNS+ 双引擎优化后，输出带时间窗的每日行程，并生成 Cesium 3D 可视化路线图。

## 三、核心模块说明

### 3.1 引擎层 (`src/engine/`)

双引擎串行架构，SA 负责粗粒度搜索（分组与聚类），VNS+ 负责细粒度精炼。

| 模块 | 职责 | 关键优化技术 |
|------|------|-------------|
| `sa.py` | 模拟退火求解单日路线 | 压缩退火（初期容忍惩罚项）、时间窗引导邻域、2-opt refine |
| `vns.py` | 变邻域搜索精炼 | 加权算子选择（自适应）、时间窗感知扰动、精英池 |
| `clustering.py` | 多种聚类方法分组 | 内置多种聚类，SA 搜索最优 k + 最优方法 |
| `fitness.py` | 适应度计算 | Numba JIT 加速，时窗惩罚 + 距离加权 |
| `search.py` | SA + VNS+ 串联 | SA 搜索最优分组 → 各日 SA 求解 → VNS+ 逐日精炼 |
| `pipeline.py` | 主流程编排 | 数据加载 → 引擎调度 → 结果格式化 → Cesium 生成 |

### 3.2 Agent 层 (`src/agent/`)

基于 LLM 的智能代理，负责自然语言交互。

| 模块 | 职责 |
|------|------|
| `planner.py` | 用户意图解析、规划流程编排 |
| `commentator.py` | 基于 ChromaDB 知识库的路线评论生成 |
| `tools.py` | 高德 API、知识库检索、引擎调度等工具集合 |

### 3.3 数据层 (`src/data/`)

| 模块 | 职责 |
|------|------|
| `amap_loader.py` | 高德 POI 搜索、驾车路径规划、营业时间解析 |
| `cesium_utils.py` | 路线 → Cesium 3D HTML 生成（Polyline 解码、轨迹渲染） |
| `knowledge_base.py` | ChromaDB 知识库管理（向量存储、语义检索） |

### 3.4 API 层 (`src/api/`)

| 模块 | 职责 |
|------|------|
| `routes.py` | FastAPI REST 端点 |
| `schemas.py` | Pydantic 请求/响应模型定义 |

### 3.5 前端 (`frontend/`)

| 模块 | 职责 |
|------|------|
| `app.py` | Streamlit 应用入口 |
| `components/` | UI 组件（日计划卡片、路线详情等） |
| `static/` | 静态资源（CesiumJS 库、生成的 HTML） |

## 四、数据流

```
用户输入（自然语言）
    │
    ▼
Agent 解析意图 ──→ 高德 API (POI搜索 + 驾车路径规划)
    │
    ▼
构建成本矩阵 + 景点信息
    │
    ▼
┌─────────────────────────────────────┐
│          SA 阶段                     │
│  遍历聚类方法 × k 值组合            │
│  对每组解调用 SA 求解               │
│  选出最优 (方法, k)                 │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│        VNS+ 阶段                     │
│  以 SA 解为初始解                   │
│  逐日运行 VNS+ 精炼                 │
│  输出最终路线                       │
└──────────────┬──────────────────────┘
               ▼
结果格式化 → Cesium 3D HTML 生成
    │
    ▼
Streamlit 展示 + LLM 评论
```

## 七、关键设计决策

### 双引擎架构（SA → VNS+）

SA 擅长全局搜索（搜索最优聚类 k + 方法），VNS+ 擅长局部精炼。串联后用 VNS+ 在 SA 解上二次优化，兼顾搜索广度与深度。

### 压缩退火（Compressed Annealing）

SA 阶段使用压缩退火：初期容忍惩罚项，允许探索不可行区域；后期逐步恢复真实成本，强制收敛到可行解。

### Numba JIT 加速

适应度计算是算法最热点路径，使用 `@njit(cache=True)` 编译为机器码，避免 Python 解释开销。

### LLM Agent 封装

将高德 API 调用、引擎调度、知识库检索封装为 Agent 工具，用户以自然语言交互，降低使用门槛。
